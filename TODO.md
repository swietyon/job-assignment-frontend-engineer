I’ve managed to implement most of the core requirements of the assignment, including the complete Profile page with the author's image, bio, name, and their specific list of articles. In the Article view, the Markdown text correctly converts to HTML, the routing is set up with dedicated links for authors and articles, and the logout function successfully clears the session and redirects users back to the homepage. The only piece left unfinished from the checklist is the actual functionality for the favorite counter.

Looking at the codebase from an architectural perspective, there are quite a few things I would improve if I had more time. First of all, I would definitely move the API base URLs into a proper .env file with the correct prefixes, as keeping them hardcoded across multiple files is a bit messy and makes maintenance difficult. I would also focus on refactoring the larger page structures, like the Profile and Article components, by breaking them down into smaller, more manageable sub-components to drastically improve readability and code reusability.

Another critical area I would address is session security and state management. I would implement proper token validity checks, potentially using global interceptors, to handle expired tokens gracefully instead of letting the app fail silently on restricted actions. When it comes to the followers and favorites counters, the ideal solution would be to handle this logic directly on the backend database. However, as a temporary frontend workaround to keep the counter states perfectly synchronized across different views instantly, I would look into integrating a global state manager like Redux or a cleaner React Context setup.

Finally, I would strongly advocate for upgrading the entire project to the latest version of React. Working with version 17 means the application is missing out on a lot of modern performance optimizations, better ecosystem support, and cleaner hooks. Modernizing the tech stack would not only make the application much more future-proof but also significantly easier to scale and maintain in a real production environment.
